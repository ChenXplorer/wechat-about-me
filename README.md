# wechat-about-me

> 用你自己的微信聊天记录，产出一份几万字的《About Me》行为画像报告。

这是一份给 **Claude Code**（或其他长上下文 LLM Agent）用的执行指南。把 [GUIDE.md](./GUIDE.md) 交给你的 Agent，再告诉它"我的微信聊天记录在 `<路径>`"，它会按六阶段流程跑完整个分析，最终产出一份 `about_me.md`。

没有聊天记录也没关系——Agent 会先引导你用 weflow 导出。

---

## 它能做什么

- 基于 **行为层面的模式**（不是自我填表）理解自己：在什么场景下更话多、对谁回复更快、每年关注点如何迁移
- 识别 **Skill 三层辐射图**：你教过的 = 真掌握；别人问你的 = 他人眼里的专长；词汇熟练度 = 不经意领域；错位处最有价值
- 找出 **盲区和内部矛盾**：每个优点配一个阴影面，每个缺点配可操作动作
- 做出一份 **严肃报告风**的长文档（不是星座测评，不是 DISC，不出 MBTI 4 字母结论）

## 它做不到什么

- 因果关系（只能看相关性和时序）
- 心理诊断（不会给你贴"回避型依恋""高敏感人群"这种标签）
- 对方视角（你只看得到自己这端的数据）
- 没发生在微信里的事情（电话、面对面、邮件、其他 IM）

---

## 快速开始

**前提**：有一个能访问 Claude 模型的 Claude Code 环境。

1. 把这个仓库的 [GUIDE.md](./GUIDE.md) 下载到本地
2. 打开 Claude Code，在你想放项目的目录开始对话
3. 发这段话：

   > `@GUIDE.md` 帮我实现这个流程，我的微信聊天记录在 `<绝对路径>`（如果还没导出，就写"还没导出"）

4. Agent 会按流程推进：
   - 没导出 → 它会引导你用 weflow 导出
   - 导出了 → 它从阶段 0 顺序执行，每个阶段停下来让你确认 checkpoint
   - 最终产出 `final/about_me.md`

用户在整个流程中需要参与的时刻主要是：
- 告诉 Agent 你在群聊里常用的昵称
- 看几次 checkpoint 数据确认没有 bug
- 遇到"这个名字是谁"时确认身份（朋友/同事/家人）

## 核心设计原则（为什么这个流程不会变成星座测评）

1. **两层架构**：全量消息在本地 Python 处理（不出本机），LLM 只吃筛选后的"高信号"子集（全量的 1-3%）
2. **不引用聊天原文**：用统计 + 抽象归纳做证据（隐私 + 防止停留在表层 + 强制归纳）
3. **高保真度加权**：深夜 / 1v1 / 长对话后段 / 含决策情绪密度高的消息优先
4. **Skill 三层辐射图**：教过的 / 被问的 / 词汇熟练度三个维度独立识别能力，错位处最有洞察
5. **MBTI 4 维独立反推**：每个维度列证据和倾向，**不输出 4 字母结论**
6. **每个优点配阴影面**：没有纯粹的优点，强制配对防止讨好文案
7. **盲区不软化**：LLM 天然倾向补一句"但这也是你的可爱之处"——必须明确禁止
8. **Layer 0 写作硬规则**：9 条不可违背的规则嵌入每章 prompt，防止产出变成情感营销文案

完整原则见 [GUIDE.md 第 5 节](./GUIDE.md#5-设计原则背后的为什么遇到边界情况时判断依据)。

## 想要不同风格的分析？

这份 GUIDE 的默认风格是 **严肃报告风 + 高管教练口吻**：不引用聊天原文、不做心理诊断、不出 MBTI 4 字母结论、不用星座、不贴人格标签名。**这只是一种风格选择**——如果你想要别的输出风格，fork 这个仓库、调整 GUIDE.md 里的"Layer 0 写作硬规则"（第 4 节）即可：

| 你想要的风格 | 怎么改 GUIDE.md |
|---|---|
| **MBTI 4 字母结论**（INTJ/ENFP 等） | 删除 Layer 0 硬规则第 3 条；在阶段 3 的 `mbti_dims.json` 任务里要求输出综合判断 |
| **星座/占星对照** | 删除硬规则第 4 条；在阶段 4 加一章"星座行为对照"，让 Agent 把行为模式和你的星座特质做比对 |
| **命理 / 塔罗风** | 去掉"严肃报告风"约束，加上你想要的叙事风格；第 9 终章改为占卜式推演 |
| **DISC / 九型人格 测评风** | 删除硬规则第 5 条；在阶段 3 加一类标签打分任务，用 DISC/九型语言输出 |
| **纯心灵成长 / 鸡汤风** | 删除硬规则第 7、8、9 条（阴影面、盲区不软化、反导师腔）；改为温暖鼓励口吻 |
| **短版快照**（1-2 页） | 阶段 4 章节从 8 章压到 2-3 章；要求每章 ≤ 500 字 |
| **给特定人（比如伴侣）的版本** | 保留第 3、4、7、8 章，删除第 2 章（社交图）；加上致对方的序章 |

改动主要集中在两处：
1. **Layer 0 写作硬规则**（决定每章不能写什么、必须写什么）
2. **阶段 4 章节结构**（决定报告有哪些章）

其他设计（两层架构、高保真度加权、Skill 三层辐射、分批处理）是通用的方法论骨架，一般不用动。

> ⚠️ **提示**：这份 GUIDE 默认禁用星座/MBTI 4 字母这类标签，是因为它们在作者看来缺乏数据支撑、容易把分析变成占星。但如果你主观就喜欢这类框架——它们在心理探索上仍有参考价值——请自由定制。"适合你"比"客观严谨"更重要。

## 隐私

- 全量消息在**你本机**处理，不出网
- 筛选后的"高信号"子集（通常是全量的 1-3%）会随你的对话发给 Claude 做语义分析
- Anthropic 默认策略：数据 30 天保留期、不用于训练（以官方条款为准）
- 如果你对此不接受，不建议做这件事——用更弱的本地模型很难产出可用质量

Agent 在开始前会主动告知这一点并等你确认。
